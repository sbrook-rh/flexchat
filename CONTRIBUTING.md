# Contributing to Chat Starter

Thank you for considering contributing to Chat Starter! This document provides guidelines for contributing to the project.

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Run the setup script: `./setup.sh`
4. Make your changes
5. Test your changes: `./start.sh`
6. Submit a pull request

## Code Style

- Use consistent indentation (2 spaces for JS/JSX, 4 spaces for Python)
- Follow existing naming conventions
- Add comments for complex logic
- Keep functions small and focused

## Testing

Before submitting changes:
- Test the chat functionality works correctly
- Verify all three services start without errors
- Check that the knowledge base loads properly
- Test with different types of queries

## Adding New Features

When adding new features:
1. Update the README.md with new configuration options
2. Add example configurations if applicable
3. Update the API documentation
4. Test thoroughly

## Reporting Issues

When reporting issues, please include:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details (OS, Node version, Python version)
- Error messages or logs

## Questions?

Feel free to open an issue for questions or discussions!
